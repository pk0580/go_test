# Учебный проект: Laravel + Go (Высоконагруженная рассылка сообщений)

## Описание проекта
Данный проект предназначен для изучения интеграции Go в экосистему Laravel для решения задач, требующих высокой производительности. Основная цель — реализовать систему рассылки уведомлений (Email/Push/SMS) большому количеству пользователей.

### Почему Go?
Laravel отлично подходит для бизнес-логики, но PHP может быть медленным при обработке тысяч параллельных сетевых запросов или тяжелых вычислений. Go берет на себя роль высокопроизводительного воркера.

---

## Архитектура системы

### Компоненты
1.  **Laravel (Web/API)**:
    *   Управление пользователями, контентом сообщений и кампаниями.
    *   Создание задач на рассылку.
    *   Админ-панель и API для внешних систем.
2.  **Go Service (Sender)**:
    *   Потребление задач из очереди (Redis/RabbitMQ).
    *   Асинхронная отправка сообщений с использованием Goroutines.
    *   Контроль лимитов (Rate limiting).
3.  **Инфраструктура**:
    *   **MySQL**: Основная БД для Laravel.
    *   **Redis**: Очередь сообщений (Broker) и кэш.
    *   **Mailpit/Smtp4dev**: Для тестирования отправки почты.
    *   **Nginx**: Проксирование запросов.

### Схема взаимодействия
`Laravel (Producer)` -> `Redis (Queue)` -> `Go Service (Consumer)` -> `Внешние API / SMTP`

## Шаги разработки

### Шаг 1: Развертывание инфраструктуры (Docker)
1.  Настройка `docker-compose.yml` с сервисами: `app` (PHP-FPM), `web` (Nginx), `db` (MySQL), `redis`, `go-sender`.
2.  Создание Dockerfile для PHP (с расширениями pdo, redis).
3.  Создание Dockerfile для Go (multi-stage build).

### Шаг 2: Инициализация Laravel
1.  Установка Laravel внутри контейнера.
2.  Настройка подключения к БД и Redis.
3.  Создание модели `Message` и миграции (текст, статус, получатель).
4.  Реализация контроллера для постановки задачи в очередь Redis.

### Шаг 3: Разработка Go-сервиса
1.  Инициализация модуля (`go mod init`).
2.  Подключение к Redis (библиотека `go-redis`).
3.  Реализация воркера, который слушает очередь.
4.  Использование `sync.WaitGroup` и каналов для параллельной обработки.
5.  Имитация отправки (mock) или интеграция с SMTP.

### Шаг 4: Взаимодействие
1.  Согласование формата JSON-сообщения между Laravel и Go.
2.  Логирование результатов обработки Go-сервисом обратно в БД или через события.

### Шаг 5: Оптимизация и тестирование
1.  Тестирование производительности (нагрузочные тесты).
2.  Настройка пула воркеров в Go.
3.  Мониторинг ресурсов.

## Рекомендации по обучению
*   **Изучите Concurrency в Go**: Поймите разницу между Goroutines и обычными потоками.
*   **Формат данных**: Используйте JSON для простых задач, или Protocol Buffers для более сложных/высоконагруженных.
*   **Graceful Shutdown**: Реализуйте в Go корректное завершение работы, чтобы не терять сообщения из обработки при перезагрузке контейнера.
*   **Очереди**: Начните с Redis (RPOPLPUSH или Streams), затем попробуйте RabbitMQ для более сложных сценариев.
