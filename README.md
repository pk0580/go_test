# Laravel + Go: Высоконагруженная система рассылки сообщений

Данный проект представляет собой гибридную архитектуру, объединяющую гибкость **Laravel** для управления бизнес-логикой и высокую производительность **Go** для параллельной обработки и отправки уведомлений.

## 🚀 О проекте

Основная цель системы — обеспечить надежную и быструю доставку сообщений (Email, SMS, Push) большому количеству получателей. Laravel принимает запросы через API, сохраняет их в базе данных и ставит в очередь, а специализированный микросервис на Go обрабатывает эту очередь в многопоточном режиме.

### Основные возможности
*   **Гибридная архитектура**: Laravel для API и администрирования, Go для высокопроизводительных воркеров.
*   **Масштабируемость**: Go использует Goroutines для одновременной отправки тысяч сообщений.
*   **Надежность**: Использование Redis в качестве брокера сообщений.
*   **Мониторинг**: Встроенная поддержка Prometheus и Grafana для отслеживания производительности в реальном времени.
*   **Тестирование**: Интеграция с Mailpit для отладки отправки почты без риска реальной отправки.

## 🛠 Стек технологий

*   **Backend API**: Laravel 12 (PHP 8.2+)
*   **Worker Service**: Go (Golang)
*   **Database**: MySQL 8.0
*   **Message Broker / Cache**: Redis
*   **Monitoring**: Prometheus & Grafana
*   **Mail Testing**: Mailpit
*   **Infrastructure**: Docker & Docker Compose

## 🏗 Архитектура и алгоритмы рассылки

Система поддерживает два режима работы, которые переключаются в настройках Laravel (`SEND_MODE` в `.env`):

### 1. Асинхронная рассылка (Redis Queue) — режим `redis`
Основной высокопроизводительный режим для массовых уведомлений.
*   **Laravel (Producer)**: Принимает запрос, сохраняет в MySQL и отправляет задачу в очередь Redis (`messages_queue`).
*   **Go Sender (Consumer)**: Постоянно слушает Redis (команда `BLPop`), при появлении задачи запускает горутину из пула воркеров.
*   **Отправка**: Go-сервис выполняет физическую отправку через SMTP (Mailpit) и обновляет статус в MySQL.

### 2. Синхронная рассылка (gRPC) — режим `grpc`
Используется для мгновенной отправки с получением немедленного ответа от сервиса.
*   **Laravel**: Формирует Protobuf-сообщение и вызывает удаленный метод `SendEmail` через gRPC клиент.
*   **Go Sender (gRPC Server)**: Принимает запрос по протоколу HTTP/2, мгновенно обрабатывает его и возвращает результат (ID сообщения или ошибку).
*   **Преимущества**: Отсутствие задержек очереди, строгая типизация контрактов (proto-файлы), немедленное подтверждение доставки.

Подробное описание путей данных и реализации можно найти в [ALGORITM.md](./ALGORITM.md).

## 📈 Мониторинг и метрики

Система обеспечивает полную наблюдаемость (observability) через связку Prometheus + Grafana.

### Основные метрики:
*   **Go-сервис**:
    *   `go_sender_messages_processed_total`: Общее количество обработанных сообщений (с разделением по статусам `sent`/`failed`).
    *   `go_sender_active_workers`: Количество одновременно работающих горутин.
    *   `go_sender_message_processing_duration_seconds`: Время обработки одного сообщения.
*   **Laravel**:
    *   `laravel_messages_status_count`: Статистика по статусам сообщений в базе данных (`pending`, `sent`, `failed`).

### Как пользоваться:
1.  **Prometheus** ([http://localhost:9090](http://localhost:9090)): Сбор и хранение метрик. Можно выполнять прямые PromQL запросы.
2.  **Grafana** ([http://localhost:3000](http://localhost:3000)): Визуализация. Логин/пароль: `admin/admin`. Источник данных преднастроен.
3.  **Логирование**: Все сервисы пишут структурированные JSON-логи в `stdout`. Просмотр: `docker-compose logs -f`.

Полное руководство по мониторингу и проверке работоспособности доступно в [MONITORING.md](./MONITORING.md).

## 🚦 Быстрый старт

### Требования
*   Docker и Docker Compose
*   Git

### Запуск
1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd go_test
   ```

2. Запустите инфраструктуру:
   ```bash
   docker-compose up -d
   ```
3. Если не получилось установить pecl grpc  
   - для WSL нужно сделать конфиг файл .wslconfig в домашней директории юзера (Users/{user}) 
       с содержимым:
       ```
       [wsl2]
       memory=8GB
       processors=4
       swap=8GB
       localhostForwarding=true

      ```
   - затем перезапустить WSL (```wsl --shutdown``` и ```wsl```)
   - затем запустить:
      ```bash
         docker-compose exec app sh -c "pecl install --force grpc-1.70.0"
      ```
4. Установите зависимости Laravel (при первом запуске):
   ```bash
   docker-compose exec app composer install
   docker-compose exec app php artisan migrate
   ```
   
5. Команда для входа в контейнер приложения:
   ```bash
   docker-compose exec app sh
   ```

### Доступ к сервисам
*   **API (Laravel)**: [http://localhost:8080](http://localhost:8080)
*   **Prometheus**: [http://localhost:9090](http://localhost:9090)
*   **Grafana**: [http://localhost:3000](http://localhost:3000) (admin/admin)
