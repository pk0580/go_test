# Laravel + Go: Высоконагруженная система рассылки сообщений

Данный проект представляет собой гибридную архитектуру, объединяющую гибкость **Laravel** для управления бизнес-логикой и высокую производительность **Go** для параллельной обработки и отправки уведомлений.

## 🚀 О проекте

Основная цель системы — обеспечить надежную и быструю доставку сообщений (Email, SMS, Push) большому количеству получателей. Laravel принимает запросы через API, сохраняет их в базе данных и ставит в очередь, а специализированный микросервис на Go обрабатывает эту очередь в многопоточном режиме.

### Основные возможности
*   **Гибридная архитектура**: Laravel для API и администрирования, Go для высокопроизводительных воркеров.
*   **Масштабируемость**: Go использует Goroutines для одновременной отправки тысяч сообщений.
*   **Надежность**: Использование Redis в качестве брокера сообщений.
*   **Мониторинг**: Встроенная поддержка Prometheus и Grafana для отслеживания производительности в реальном времени.
*   **Тестирование**: Интеграция с Mailpit для отладки отправки почты без риска реальной отправки.

## 🛠 Стек технологий

*   **Backend API**: Laravel 12 (PHP 8.2+)
*   **Worker Service**: Go (Golang)
*   **Database**: MySQL 8.0
*   **Message Broker / Cache**: Redis
*   **Monitoring**: Prometheus & Grafana
*   **Mail Testing**: Mailpit
*   **Infrastructure**: Docker & Docker Compose

## 🏗 Архитектура

Система работает по принципу **Producer-Consumer**:

1.  **Laravel (Producer)**:
    *   Принимает HTTP POST запрос на эндпоинт `/api/messages`.
    *   Валидирует данные и создает запись в MySQL со статусом `pending`.
    *   Отправляет JSON-полезную нагрузку в очередь Redis (`messages_queue`).

2.  **Go Sender (Consumer)**:
    *   Постоянно слушает очередь Redis.
    *   При появлении задачи запускает воркера (используется пул воркеров).
    *   Имитирует/выполняет отправку сообщения.
    *   Обновляет статус сообщения в MySQL и собирает метрики.

3.  **Мониторинг**:
    *   Оба сервиса экспортируют метрики для Prometheus.
    *   Grafana визуализирует данные (количество отправленных сообщений, время обработки, активные воркеры).

## 🚦 Быстрый старт

### Требования
*   Docker и Docker Compose
*   Git

### Запуск
1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd go_test
   ```

2. Запустите инфраструктуру:
   ```bash
   docker-compose up -d
   ```
3. Если не получилось установить pecl grpc  
   - для WSL нужно сделать конфиг файл .wslconfig в домашней директории юзера (Users/{user}) 
       с содержимым:
       ```
       [wsl2]
       memory=8GB
       processors=4
       swap=8GB
       localhostForwarding=true

      ```
   - затем перезапустить WSL (```wsl --shutdown``` и ```wsl```)
   - затем запустить:
      ```bash
         docker-compose exec app sh -c "pecl install --force grpc-1.70.0"
      ```
4. Установите зависимости Laravel (при первом запуске):
   ```bash
   docker-compose exec app composer install
   docker-compose exec app php artisan migrate
   ```
   
5. Команда для входа в контейнер приложения:
   ```bash
   docker-compose exec app sh
   ```

### Доступ к сервисам

*   **Prometheus**: [http://localhost:9090](http://localhost:9090)

*   **Mailpit**: [http://localhost:8025](http://localhost:8025)

## 📖 API Эндпоинты

### Отправка сообщения
`POST /api/messages`
```json
{
  "recipient": "user@example.com",
  "content": "Текст вашего сообщения"
}
```

### Список сообщений
`GET /api/messages` — возвращает список всех сообщений и их текущие статусы.
